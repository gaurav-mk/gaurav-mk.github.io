Communication Protocol

How SPI Works on the LMX1205

The LMX1205 is controlled entirely through its 3 wire SPI port (SCK, SDI, CS). You send a 24-bit word while CS is held LOW, MSB first, and the chip latches each bit on the SCK rising edge (CPOL=0, CPHA=0). The maximum clock speed is 20 MHz. All three SPI pins require a 200 Ω series resistor on the board to prevent ringing.

SPI Timing — 24-Bit Write Transaction
Fig 1 — CS falls low, then 24 bits arrive MSB first: 1 R/W bit (0=write), 7 address bits (A6:A0), 16 data bits (D15:D0). CS rises to latch.

CPOL (Clock Polarity) defines the quiescent state of SCK when the bus is idle, The level the line rests at between transactions. CPHA (Clock Phase) determines which clock edge the receiver uses to latch each incoming bit, the first (leading) edge, or the second (trailing) edge. These two binary parameters combine into four distinct SPI modes. The LMX1205 requires Mode 0 exclusively, SCK idles low (CPOL=0), and each bit is captured on the rising edge (CPHA=0).

Frame Breakdown

The 24-Bit SPI Frame

Three bytes, always. Bit 23 is R/W (0 = write, 1 = read back via MUXOUT). Bits 22–16 are the 7-bit register address. Bits 15–0 are the 16-bit data word. Always write all 16 bits, including reserved fields set to their datasheet-specified reset values.

BitsFieldDescription
[23]R/W0 = Write register, 1 = Read back (data returned on MUXOUT pin, not a dedicated SDO)
[22:16]Address7-bit register address A6:A0. R0 = 0x00, R27 = 0x1B, R37 = 0x25. Unimplemented addresses have no effect.
[15:0]Data16-bit register value D15:D0. Always send all 16 bits, reserved fields must match their reset value to avoid undefined behaviour.
