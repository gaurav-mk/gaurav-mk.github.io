Clocking JESD204B/C Systems

How JESD204B/C clocking achieves deterministic data sampling between multiple ADCs, DACs, and logic devices — from subclass selection to SYSREF timing margins.

JESD204B/C SYSREF Determinism Clock Skew Texas Instruments
01 · Introduction

Why JESD204B/C clocking?

High-speed data converter systems — ADCs, DACs, and FPGAs working in concert — need every device to agree on a common time reference. Without that, each device resets its internal counters at slightly different moments on power-up, and the latency between samples becomes unpredictable from boot to boot.

JESD204B/C clocking solves this by adding a second signal, SYSREF, alongside the familiar device clock. SYSREF provides a system-wide time reference that forces every device to reset its internal dividers to the same state simultaneously — making the data pipeline latency deterministic across power cycles and across all devices in the system.

Key insight: JESD204C allows higher lane rates (up to 32 Gbps vs 12.5 Gbps for B) but the clocking architecture — device clock + SYSREF — is identical between B and C. Everything on this page applies equally to both.
02 · Standard Background

What is JESD204?

JESD204 is a JEDEC standard that defines a serialized high-speed interface between data converters (ADCs, DACs) and logic devices (FPGAs, ASICs). Instead of parallel buses with strict trace-length matching, it uses a small number of serial lanes — dramatically simplifying PCB layout.

The jump from JESD204A to JESD204B was the critical one: it introduced deterministic latency via SYSREF-based synchronization (Subclass 1) and the multi-frame clock (LMFC). JESD204C kept the same clocking model and bumped the lane rate ceiling.

03 · Architecture

Traditional clocking vs. JESD204B

Traditional

Each converter and the logic device receive a single clock. Data is moved over many parallel lanes with tight trace-length matching. Synchronization is left entirely to the user.

ADC ──────── many parallel data lanes ──→ FPGA
DAC ←─────── many parallel data lanes ─── FPGA
             ↑ tight length matching required ↑
  • Many parallel PCB traces
  • Strict trace length matching
  • Non-deterministic latency on power-up
JESD204B/C

Each converter and the logic device receive a device clock + SYSREF. Data travels over a few high-speed serial lanes. No trace length matching needed.

ADC ── 1–4 serial JESD lanes ──→ FPGA
DAC ←─ 1–4 serial JESD lanes ─── FPGA
            ↑ no length matching ↑
SYSREF ────────────────────────→ all
  • Few high-speed serial lanes
  • No length matching required
  • Deterministic latency every power-up

The internal clocks inside each JESD204B device — Frame Clock, Local Multi-Frame Clock (LMFC) — are all derived by dividing the device clock. SYSREF instructs every device to reset these dividers simultaneously. Once aligned, the user can know the exact time between any two samples anywhere in the system to within one LMFC period.

Analogy Think of SYSREF as the starter's gun at a race. Every runner (device) has been warming up (running their internal clocks) independently. The gun fires (SYSREF edge arrives) and everyone resets to the same starting block. From then on, the whole system is synchronized.
04 · Subclass Selection

JESD204B Subclasses

JESD204B defines three subclasses that differ in how deterministic latency is achieved. Only Subclasses 1 and 2 provide determinism; Subclass 0 is essentially backward-compatible legacy mode.

Subclass 0 Legacy
No support for deterministic latency. Backward compatible with JESD204A. Use only when determinism is not required.
Subclass 2 Less common
Uses the ~SYNC signal (which also requests link initialization) as the timing reference. Not usable at higher frequencies and less popular than Subclass 1.
Subclass 1 Recommended
Uses an external SYSREF signal as the timing reference. Works at all frequencies, supports continuous and one-shot SYSREF modes. The industry standard choice for high-speed systems.
This page focuses on Subclass 1. The LMX1205 is a Subclass 1 clocking device — it generates device clocks and SYSREF outputs with programmable delays to precisely position the SYSREF edge inside each receiver's valid window.
05 · Subclass 1

Subclass 1 Operation

The mechanism is elegantly simple: after a rising edge of SYSREF, the very next rising edge of the device clock becomes a universal time reference point. Every device in the system resets its internal frame clock and LMFC dividers at that edge.

Subclass 1 Timing Diagram
After SYSREF rises, the next device clock rising edge (dashed line) resets all internal LMFC dividers. Every device in the system lands on this same edge.

Sometimes more than one SYSREF edge is required to reset all elements for phase determinism — especially when multiple JESD204B devices need to be aligned. The clocking device can be programmed to send exactly the right number of pulses.

LMFC = Local Multi-Frame Clock. Derived by dividing the device clock by K×F (where K = frames per multiframe, F = octets per frame). SYSREF resets this divider so all devices in the system share the same LMFC phase.
06 · SYSREF Modes

Types of SYSREF signals

There are three ways to deliver SYSREF to the system, each with different trade-offs between noise, convenience, and control:

Continuous SYSREF

SYSREF toggles continuously at a subharmonic of the device clock. Good for debug and measurement. Causes radiated spurious noise and extra power — not ideal for production.

One-Shot SYSREF

Single (or few) pulses sent only when a resync is needed. Minimizes radiated noise. May require re-programming receiver registers between pulses to mask different JESD204B elements.

Gapped Periodic SYSREF

Periodic bursts with gaps. Subsequent one-shots are guaranteed to align to the same LMFC timing — essential when reprogramming receiver elements between pulses.

AC coupling consideration: When SYSREF is AC coupled, a single pulse won't DC-balance the signal. Continuous SYSREF must be used temporarily during initialization — send it, wait for DC balance, let the receiver synchronize, then turn it off.
07 · Why Use It

Design benefits of JESD204B/C

Fewer PCB traces. Traditional interfaces need dozens of parallel data lanes, all length-matched to picosecond accuracy. JESD204B replaces them with 1–8 high-speed serial lanes with no length matching required.
Same device clock for all converters. Each device's internal dividers reduce the shared device clock to whatever sample rate the converter needs. This eliminates the need for multiple clock frequencies at the clocking IC output.
Reduced cross-talk. Fewer clock signals at different frequencies means less opportunity for inter-clock interference on the PCB and inside the clocking IC.

08 · Core Concept

Determinism & the SYSREF valid window

For Subclass 1 determinism to work, SYSREF must "mark" the same device clock rising edge every time — across power cycles, temperature changes, and board-to-board variation. The way this is guaranteed is by ensuring the SYSREF rising edge falls inside the SYSREF valid window of every receiving device.

SYSREF Valid Window
The SYSREF valid window = Device clock period − Setup time − Hold time. The SYSREF edge must land inside this window to deterministically mark the same device clock edge every time.
SYSREF valid window = Tdevice_clock − tsetup − thold
Where setup and hold times are found in the converter's datasheet

The maximum setup time is typically approximated as the device clock period minus the hold time. Both setup and hold times are specified per-device in the converter's datasheet and vary with supply voltage, temperature, and process.

09 · Timing Violations

What happens if timing is missed? (Device skew)

If the SYSREF edge lands outside the valid window — violating setup or hold — a different device clock edge gets marked. The device sees its LMFC divider reset one full clock cycle too early or too late. From the system's perspective this looks like a 1-cycle digital skew on that device.

Missed Timing Scenarios
When SYSREF is late (outside the valid window on the high side), the +1 cycle device clock edge gets marked instead of the intended ±0 edge — introducing a 1-cycle determinism error.

Two delay adjustment mechanisms exist to move the SYSREF edge into the valid window:

1
Digital delay — Coarse adjustment. Step size equals the half-period of the VCO clock driving the SYSREF output divider. For a 3 GHz VCO: ½ × (1/3 GHz) ≈ 166.7 ps per step. Very repeatable over temperature and voltage.
2
Analog delay — Fine adjustment. Typically 10–150 ps per step. Less repeatable than digital delay — accuracy degrades over temperature and voltage. Used to fine-tune after digital delay has coarsely positioned the edge.
Device clock timing adjustments are used to minimize skew between device clocks at different converters. SYSREF timing adjustments are used to position the SYSREF edge inside the valid window. These are independent knobs.
10 · Margin Calculations

SYSREF timing margin examples

The goal is to select a digital delay setting where the SYSREF edge sits as far from both edges of the valid window as possible. The margin you achieve depends on how the valid window compares to the digital delay step size.

Example 1 — Valid window > 3× delay step
Parameters
VCO frequency: 3000 MHz
Device clock: 1500 MHz
Clock period: 666.7 ps
Setup time: 50 ps
Hold time: 50 ps
Valid window: 566.7 ps
Delay step: 166.7 ps
Result
3 delay steps fit inside the valid window. Select the middle one — margin is at least 1 full delay step size (166.7 ps). Provided the SYSREF routing skew is less than one step size, determinism is guaranteed.
Example 2 — Valid window < 3× delay step
Parameters
Valid window: 300 ps
Delay step size: 200 ps
Margin = (300 − 200) / 2 = 50 ps
Result
Worst case: only 50 ps margin. The system will be deterministic only if routing skew between device clock and SYSREF is less than 50 ps. Consider adding analog delay or improving routing to increase margin.
When valid window < 3 × delay step:
Margin = (SYSREF valid window − delay step size) ÷ 2

Use min/max delay step sizes to account for non-uniform step sizes across corners.
11 · Hardware

Clock interface: AC vs DC coupling

The physical electrical interface between the clocking IC and each converter matters — especially for SYSREF, where single pulses need to faithfully transfer without distortion.

DC Coupled

Simplest approach for SYSREF. Allows single pulses to transfer cleanly. Common mode must be matched manually using resistor dividers (Rs, Rb) at the receiver input.

LVDS: direct 50Ω trace → receiver
LVPECL: use Rs (series) + Rb (to GND)
to voltage-divide to correct common mode
  • Works with one-shot SYSREF
  • Must manage common mode voltage
  • Simpler synchronization procedure
AC Coupled

Eliminates common-mode matching. But a single SYSREF pulse won't DC balance — you need a special initialization procedure.

1
Turn on continuous SYSREF
2
Wait for AC lines to reach DC balance
3
Allow receiver to sync (toggle register if needed)
4
Turn SYSREF off

In practice, DC coupling is preferred for the SYSREF path in Subclass 1 systems because it enables clean one-shot operation. AC coupling is common for the device clock path (where the signal is always continuous) since it removes common-mode voltage constraints.
