TICS Pro Interface — LMX1205 Explained

Every panel, dropdown, button, and number in the Texas Instruments TICS Pro GUI for the LMX1205 decoded block-by-block — no prior RF or tool experience needed.

TICS Pro GUI LMX1205 6 Blocks Explained SYSREF Clock Configuration
TICS Pro (Texas Instruments Clocks and Synthesizers Pro) is TI's free GUI tool for configuring clock chips like the LMX1205 over SPI — either through a USB-connected evaluation board, or by exporting the generated register map for use in firmware. Every parameter you can set via SPI appears as a labelled field or dropdown in the interface; the tool calculates frequencies and timing in real time and prevents you from entering invalid combinations.
Simple Analogy Think of TICS Pro as a mixing desk for your clock chip. Instead of turning physical knobs, you type in frequencies and hit buttons, and the tool figures out which register bits to write — then sends the correct bytes over SPI to the chip. The six panels on screen are the six functional sections of the LMX1205, each grouped by what they control.

1
Top-Left Panel
CLKIN · Multiplier · Temperature Sensor · General
Block 1 Screenshot

This is the starting point for any TICS Pro session. You tell the tool how fast your incoming clock is, choose whether to buffer, divide, or multiply it, and set up diagnostic and power controls. The pink shaded sub-panel belongs entirely to the on-chip multiplier (PLL + VCO).

Input CLKIN frequency field
Type your input clock frequency here in MHz (the screenshot shows 1280 MHz). This is the frequency of the signal arriving at the CLKIN_P/N pins from your upstream PLL or oscillator. TICS Pro uses this value to calculate every downstream frequency, step size, and divider restriction shown in the other panels. Always set this first.
Input Input delay (0.00 ps)
Adds a programmable delay of 0–60 ps (1.1 ps steps) to the CLKIN path before it reaches the frequency multiplexer. Use this to compensate for propagation delay from your upstream master PLL to this chip — e.g. if your PLL output arrives 35 ps earlier than expected due to a shorter trace, set 35 ps here to re-align the whole output section. Changes here shift all four CLKOUTs simultaneously.
Select Mode: Buffer / Div2/x2
Selects the operating mode of the main clock path (CLK_MUX register):
  • Buffer — pass-through at the same frequency (÷1). Lowest noise.
  • Div2/x2 … Div8/x8 — divides or multiplies by the selected factor. The dropdown shows both the divide and multiply options for each ratio.
The two "n/a" boxes below show intermediate calculated frequencies that appear when a divide or multiply factor is chosen.
Button Read Lock Detect
Reads the rb_LOCK_DETECT bit from the chip via SPI and displays whether the multiplier's internal VCO is Locked or Unlocked. You must wait for "Locked" before enabling outputs in multiplier mode — an unlocked VCO will produce the wrong frequency or a noisy output. The MUXOUT pin also shows this in hardware.
Button Calibrate Multiplier
Triggers the VCO calibration routine by writing to register R0. The chip's internal state machine searches for the optimal VCO frequency band and core for the requested output frequency. You must run calibration:
  • The very first time the chip powers up in multiplier mode
  • Whenever you change the multiply factor
  • Whenever you change the input frequency
Typical calibration time: ~750 µs. First-ever calibration or mode change: up to ~5 ms.
Status fSM_CLK display
Shows the calculated State Machine Clock frequency (2.50 MHz in the screenshot). This internal clock drives the multiplier calibration logic and lock detector. It is derived from CLKIN by the dividers shown below it (÷8 × ÷64 here → 1280 / 8 / 64 = 2.5 MHz).

Rules: must be <30 MHz; must be at least 2× faster than the SPI write speed; must be present for calibration and lock detection to work.
Toggle SMCLK_EN checkbox
Enables or disables the state machine clock. Keep enabled during calibration and when monitoring lock detect. Once calibration is confirmed and lock detection is not needed, you can uncheck this to shut off the state machine clock — this eliminates a potential source of spurious tones (spurs) at the state machine clock frequency on the output.
Input ÷8 × ÷64 (SMCLK dividers)
Two cascaded dividers that scale CLKIN down to fSM_CLK. The first (÷8) is SMCLK_DIV_PRE and the second (÷64) is SMCLK_DIV. TICS Pro automatically suggests valid values when you change the CLKIN frequency, but you can override them manually. Together: 1280 MHz ÷ 8 ÷ 64 = 2.5 MHz.
Status Estimated Current (968 mA)
TICS Pro calculates the approximate total supply current based on which outputs are enabled, their power level, and the operating mode. Use this to budget your 2.5 V power supply and decouple it appropriately. At full load (all outputs + SYSREFs) the LMX1205 can draw ~1130 mA; 968 mA here reflects some outputs disabled.
Button Temp Sensor — Read
Reads the rb_TEMPSENSE register from the chip and shows the raw code. Convert to temperature with: T (°C) = 0.65 × Code − 351. Useful for thermal characterisation or to diagnose overheating. The TEMPSENSE_EN checkbox must be checked for the sensor to power up.
Select MUXOUT Pin — Readback
The MUXOUT pin on the chip is a configurable diagnostic output. This dropdown sets what it reports. Options include:
  • Readback — outputs SPI register data during a readback transaction (tri-stated otherwise)
  • Lock Detect — drives HIGH when the multiplier VCO is locked
  • Other internal status signals
Toggle POWERDOWN / RESET
POWERDOWN: Puts the entire chip into a low-power standby state (~13.5 mA). All outputs go tri-stated. Registers are preserved; un-check to wake up.

RESET: Triggers a software power-on reset — all registers return to their factory defaults (buffer mode, all SYSREFs off). The bit self-clears after the first subsequent register write. Use this at startup to guarantee a clean initial state.
2
Bottom-Left / Centre Panel
SYSREF Global Delay & Per-Channel Delay Codes
Block 2 Screenshot

This panel controls the fine timing position of every SYSREF output relative to the clock outputs. It splits into two halves: a global delay generator that sets the step size (left), and per-channel delay codes that set how many steps each SYSREF output is offset (right). Getting SYSREF timing right is the key step for JESD204C Subclass 1 deterministic latency.

Simple Analogy Imagine a ruler (global delay range) and a pencil mark (per-channel code). The global panel sets how fine the ruler divisions are; the per-channel codes say how many divisions to count for each SYSREF output.
Toggle SYSREF_DLY_BYP
When unchecked (default): the phase interpolator delay generator is active — fine SYSREF delay is available via the step codes.

When checked: the delay generator is bypassed — SYSREF goes out with zero additional delay. Use bypass only if you don't need fine positioning, or to measure the raw propagation delay before applying corrections.
Select SYSREF_DLY_DIV dropdown
Sets the divider that generates the interpolator reference frequency from CLKIN. The screenshot shows "3: Div16 (6.4 to 12.8 GHz)" — meaning CLKIN is divided by 16 before entering the phase interpolator, giving an interpolator frequency in the 400–800 MHz window. The tool shows the valid range for each option so you pick the right one based on your CLKIN frequency:
  • CLKIN ≤ 0.8 GHz → Div2
  • 0.8–1.6 GHz → Div2
  • 1.6–3.2 GHz → Div4
  • 3.2–6.4 GHz → Div8
  • 6.4–12.8 GHz → Div16
Calculated fINTERPOLATOR = 80.00 MHz
The frequency of the phase interpolator clock — derived from CLKIN ÷ SYSREF_DLY_DIV. Here: 1280 MHz ÷ 16 = 80 MHz. This value must fall in the 400–800 MHz range for most CLKIN frequencies (or 200–400 MHz for low CLKIN). TICS Pro validates this and highlights it red if out of range.
Calculated tDELAY_STEPSIZE = 24.61 ps
The timing resolution of one SYSREF delay step:

StepSize = SYSREF_DLY_DIV / (fCLKIN × 508)

Here: 16 / (1280 MHz × 508) = 24.61 ps per step. At 12.8 GHz input the step size shrinks to <2.5 ps. This tells you how precisely you can position the SYSREF edge.
Calculated tDELAY_RANGE
Shows the total achievable delay range — from the minimum negative offset to the maximum positive offset in picoseconds. The screenshot shows [−6250, +6225] ps, which is ±one full interpolator period (1/80 MHz = 12.5 ns total). This confirms you can position SYSREF anywhere within ±half a DCLK period relative to the clock edge.
Input SYSREF Channel codes (0–4, each = 127)
Five spinner fields — one for each SYSREF output (SYSREF0, SYSREF1, SYSREF2, SYSREF3, LOGISYSREF). Each accepts a value from 0 to 508. The tool calculates:

Total Delay = StepSize × Code

With step 24.61 ps and code 127: delay ≈ 3125 ps. Set all five to the same value if your PCB routes all SYSREF outputs identically; adjust individually if trace lengths differ.
Select ICLK* dropdown (per channel)
Selects the reference clock each SYSREF phase interpolator uses internally. ICLK* (with asterisk) means the interpolator reference is derived from the same divider as the corresponding clock output — the recommended and most common setting. Changing this affects the phase relationship between SYSREF and DCLK; only change it if you have a specific skew requirement.
3
Left-Centre Panel
SYSREFREQ Position Measurement & Windowing
Block 3 Screenshot

This panel is about measuring and optimising how the chip captures the incoming SYSREFREQ signal. When the LMX1205 is in Repeater or Retime mode it needs to sample the external SYSREFREQ pulse at just the right moment relative to CLKIN — this panel shows the measured skew and lets the chip auto-centre its capture window.

Button Read SYSREFREQ Position
Triggers a measurement read: the chip uses its internal windowing circuit to measure the timing relationship between the SYSREFREQ input edge and the CLKIN rising edge, then reports it back via SPI. The result appears as:

SYSREF to CLK Skew = 73.1 ps

This means the SYSREFREQ edge arrives 73.1 ps before the next CLKIN edge. The windowing circuit uses this measurement to set an optimal capture timing.
Calculated SYSREFREQ_DLY Recommendation = 12
After reading the SYSREFREQ position, TICS Pro calculates the optimal delay code to program into SYSREFREQ_DLY. This code shifts the internal sampling window so that the SYSREFREQ edge lands comfortably in the centre of the capture window — maximising setup and hold margin. Write this recommended value to the chip to prevent metastability or missed-capture errors.
Status Bin / Dec register display
Shows the raw binary and decimal representation of the register word that will be written based on the current settings. The binary string 100 0011 0000 0000 0001 1000 0000 0000 1 is the complete SPI word for the SYSREFREQ windowing register. Useful for debugging: if the chip is behaving unexpectedly, compare this to the device register map in the datasheet to confirm the correct bits are set.
Toggle SYNC_STOP
When checked, stops the SYNC operation from propagating through the windowing circuit. Use this when you want to freeze the current SYSREF alignment and prevent any subsequent SYNC pulse from re-randomising the divider phases. Typically left unchecked during normal operation.
Toggle SYSWND_UPDATE_STOP
Prevents the SYSREFREQ windowing circuit from updating its internal state after a SYSREF capture event. Check this once you have a stable, valid capture to lock in the measured skew — the chip will not re-attempt to update the window position until you uncheck it. This is useful in production to ensure repeatable behaviour after the initial alignment.
4
Centre Panel
SYSREF Frequency Generator — Mode, Dividers & Trigger
Block 4 Screenshot

This is the SYSREF engine — where you choose whether the chip generates SYSREF internally, repeats an external one, or fires a one-shot pulse burst. It also contains the SYNC feature (phase-aligns all internal dividers across one or multiple devices) and the frequency divider chain that sets the SYSREF output frequency.

Simple Analogy SYSREF is like a starting pistol for your ADCs and DACs. This block decides: does the LMX1205 fire the pistol itself on a schedule (Generator), does it relay someone else's pistol shot (Repeater), or does it fire exactly once on command (Pulser)?
Select 0: Continuous (SYSREF Mode)
The main mode dropdown — sets SYSREF_MODE:
  • 0: Continuous — SYSREF pulses repeat forever at the programmed frequency. Easiest for debug; may cause crosstalk at high frequencies in production.
  • 1: Pulser — fires a burst of N pulses (set by the Pulser spinner) each time triggered. Preferred in production — zero crosstalk between alignment events.
  • 2: Repeater — buffers the SYSREFREQ input and fans it out to all outputs.
  • 3: Repeater Retime — re-times SYSREFREQ to CLKIN before distributing — cleanest edges, eliminates metastability.
Select Trigger: Force High
Controls how SYSREF generation or pulsing is triggered (SYSREFREQ_INPUT):
  • Force High — software-forces the internal trigger signal high, causing SYSREF to generate/pulse immediately without needing an external pin. Use for bench testing.
  • SYSREFREQ pin — generation is gated by the physical SYSREFREQ_P/N input — an external device sends the trigger.
Input Pulser count (1 ÷)
In Pulser mode, sets how many SYSREF pulses to fire per trigger event (1–16, set by SYSREF_PULSE_CNT). The screenshot shows 1 pulse. Most JESD204C applications need only 1 pulse to latch all converters. Some older systems need 2 or 4 to guarantee all converters see the pulse despite different propagation delays.
Toggle SYSREF_EN
Master enable for the SYSREF generation block. Must be checked for any SYSREF outputs to appear. Unchecking it powers down the entire SYSREF divider chain and output buffers — saving ~425 mA of current when SYSREF is not needed (e.g. during initial PLL lock-up before alignment is performed).
Input SYSREF Dividers: ÷4 × ÷20
Two cascaded dividers set the SYSREF output frequency:

fSYSREF = fCLKIN / (SYSREF_DIV_PRE × SYSREF_DIV)

Here: 1280 / (4 × 20) = 16 MHz. The first divider (SYSREF_DIV_PRE = ÷4) must bring the signal below 3.2 GHz before the second divider (SYSREF_DIV = ÷20, range ÷2–4095) takes it to the final frequency.

Note: odd SYSREF_DIV values do not give a 50 % duty cycle.
Calculated fSYSREF display
Shows the resulting SYSREF output frequency in MHz, calculated from the dividers above. This must be a sub-multiple of the clock output frequency, and must also be a multiple of the phase interpolator frequency when the delay generator is active (fINTERPOLATOR mod fSYSREF = 0).
Toggle SYNC_EN
Enables the SYNC feature (SYNC_EN). When triggered, SYNC simultaneously resets all internal dividers (clock output dividers, LOGICLK dividers, SYSREF dividers) to a known phase state. This is essential for multi-device synchronisation: you can tie the SYNC pin of multiple LMX1205 chips together so they all reset in unison, giving deterministic phase relationships across a whole board.
Input SYNC Delay (0 × 8 ps)
Adds a programmable delay between the SYNC event and when the dividers actually reset. Displayed as N × 8 ps. This lets you fine-tune the phase of the post-SYNC output across different chips or channels. 0 means the reset happens immediately — appropriate for most designs.
Toggle SYSWND_LATCH
Latches the result of the SYSREFREQ windowing measurement into a register so you can read it back later without the measurement being overwritten by a new SYSREFREQ event. Check this before reading the SYSREFREQ position if you want a stable snapshot.
Toggle SYSREFREQ_CLR
Clears the SYSREFREQ capture flag. After the chip detects and captures an incoming SYSREFREQ edge, it sets an internal flag. Write SYSREFREQ_CLR to reset that flag so the chip is ready to capture the next SYSREFREQ event cleanly. Analogous to clearing an interrupt in a microcontroller.
Input SYSREFREQ amplitude / P=N
Configures the expected characteristics of the external SYSREFREQ input signal (used in Repeater/Retime modes). The amplitude field (25 mV shown) sets the internal comparator threshold. P=N means positive and negative inputs are treated symmetrically (differential mode). Adjust if you are driving SYSREFREQ single-ended.

Right Panel — Repeated ×4 for CH0–CH3
Output Channel — CLKOUT Power, Delay & SYSREFOUT Settings
Block 5 Screenshot (Channel 0)

Each of the four main clock outputs (CH0–CH3) has its own identical panel. The top half controls the clock output buffer (CLKOUT); the bottom half (green/yellow background) controls the paired SYSREF output buffer (SYSREFOUT). Both halves have independent power and delay settings.

Two-level enable system: CH_EN controls the entire internal channel path (signal routing inside the chip). CLK_EN only gates the output buffer (the driver that actually drives the pin). To save the most power, disable CH_EN. To just silence the output pin while keeping the internal path alive (e.g. during SYNC), disable CLK_EN only.
Toggle CH0_EN
Powers the entire internal channel 0 path — the signal routing from the output of the frequency mux all the way to the output pin driver. When unchecked, the whole channel is powered down regardless of CLKx_EN or CLKx_PWR settings. Saves the most current. The screenshot shows this unchecked — meaning channel 0 is disabled.
Input PWR: 6 (CLKOUT drive strength)
Sets the CLKOUT output buffer drive strength (CLKx_PWR), from 1 (minimum, lowest current) to 6 (maximum, highest drive). Higher values push more current into the load, increasing the output voltage swing. At PWR = 6 and 6 GHz, the single-ended output power is approximately +4.8 dBm into 50 Ω. Lower values reduce power consumption but also reduce voltage margin at the receiver.
Input DLY: 0 (CLKOUT delay code)
Per-output phase interpolator delay code for CLKOUT0 (CLK0_DLY). Range 0–64, step size ≈0.9 ps. Set different values on CLKOUT0–3 to compensate for different PCB trace lengths to each converter. The calculated absolute delay in picoseconds appears as a tooltip or alongside the field in the tool (2708.66 ps propagation + 0 adjustment here).

Tip: Use output delay ≥ 4 for frequencies below 1.5 GHz for best noise performance.
Calculated CLK0 — 1280.0 MHz display
Shows the calculated output frequency of CLKOUT0 in MHz, derived from CLKIN and the selected mode (Buffer here → 1280 MHz = CLKIN directly). This updates automatically when you change the mode or CLKIN frequency. Use this to verify your frequency plan is correct before programming the chip.
Toggle CLK0_EN
Enables the CLKOUT0 output buffer. CH0_EN must also be enabled for any signal to appear. Disabling CLK0_EN while keeping CH0_EN on is useful during a SYNC event — the internal divider keeps running synchronously while the output is muted, preventing glitches on the PCB during the alignment sequence.
Input PWR: 4 + LP checkbox (SYSREF drive)
Sets the SYSREF0 output buffer drive strength (SYSREF0_PWR = 0–7) and enables Low Power mode (SYSREF0_PWR_LOW). In LP mode, the same PWR code delivers about half the output swing — useful when your receiver needs a lower-amplitude SYSREF or when you are trying to reduce power. The table in the datasheet maps each combination to the actual VOD swing.
Input VCM: 0.65 (SYSREF common mode)
Sets the SYSREF0 common-mode output voltage (SYSREF0_VCM) — the DC level that both SYSREF0_P and SYSREF0_N hover around. 0.65 V here. The LMX1205 is unusual in that its SYSREF outputs can be DC-coupled (unlike the CLKOUT pins which must be AC-coupled). Setting VCM correctly is critical for DC coupling — it must match the receiver's expected common-mode input level.
Input DLY: 0 (SYSREF delay, showing 2708.66 ps)
Per-channel SYSREF0 delay code, separate from the global SYSREF delay in Block 2. The total SYSREF delay for this output is:

Total = Global Steps × StepSize + (this code × StepSize)

The displayed value (2708.66 ps) is the absolute propagation delay from CLKIN to the SYSREF output pin — useful for calculating the expected CLKOUT-to-SYSREF skew at the ADC input.
Button −Edge / Min buttons
−Edge: inverts the SYSREF output polarity — flips which edge (rising or falling) is the "active" synchronisation edge. Useful if your ADC expects SYSREF on the falling edge.

Min: automatically sets the delay code to the minimum non-zero value, snapping the SYSREF edge as early as possible within the available delay range. Useful as a starting point before fine-tuning.
Calculated SYSREF0 — 16.0 MHz display
Shows the SYSREF0 output frequency: 1280 MHz ÷ (4 × 20) = 16 MHz, matching the dividers set in Block 4. This frequency is shared across all five SYSREF outputs — the per-channel delay codes in Block 2 shift the phase, not the frequency.
6
Bottom-Right Panel
Logic Clock & LOGISYSREF — FPGA-Rate Output Configuration
Block 6 Screenshot

This panel configures the two lower-frequency clock outputs — LOGICLKOUT0 and LOGICLKOUT1 / LOGISYSREFOUT — that are typically used to drive FPGAs and digital processors. They use a separate, deeper divider chain than the main CLKOUT channels so they can reach much lower frequencies (down to ≈1 MHz), while remaining phase-coherent with the main clocks.

Why a separate divider chain? The main CLKOUTs can only divide by 2–8 (maximum ÷8). An FPGA running at 16 MHz from a 1280 MHz CLKIN needs a total division of ÷80 — far beyond the main chain. The LOGICLK chain supports a 3-stage cascade (÷1,2,4 → ÷1–1023 → ÷1,2,4,8) giving a total division range up to 32,736.
Toggle LOGIC_EN
Master enable for the entire logic clock section. When unchecked, LOGICLKOUT0, LOGICLKOUT1, and LOGISYSREFOUT all go dark and the divider chain is powered down. Check this first before configuring any other logic clock field — TICS Pro may grey out the sub-controls when LOGIC_EN is off.
Input ÷4 × ÷20 (LOGICLK divider chain)
Two cascaded dividers set LOGICLKOUT0's frequency:

fLOGICLK0 = fCLKIN / (DIV_PRE × DIV)

Here: 1280 / (4 × 20) = 16 MHz.
  • LOGICLK_DIV_PRE (÷4): must bring the signal to ≤3.2 GHz — mandatory for CLKIN >6.4 GHz.
  • LOGICLK_DIV (÷20): the main divider, range ÷1–1023. Odd values give non-50% duty cycle.
Toggle LOGICLK2_EN + ÷/2
Enables the second logic clock output (LOGICLKOUT1) on the LOGISYSREFOUT pin. When enabled, the pin becomes a clock instead of a SYSREF.

The ÷/2 spinner (LOGICLK2_DIV) adds an additional divide on top of the LOGICLKOUT0 frequency — options are ÷1, ÷2, ÷4, ÷8. So if LOGICLKOUT0 = 16 MHz and LOGICLK2_DIV = ÷2, then LOGICLKOUT1 = 8 MHz. Useful for systems needing two different logic-rate clocks from the same chip.
Select FMT: LVDS (LOGICLK output format)
Sets the output driver format for LOGICLKOUT0 (LOGICLK_FMT):
  • LVDS — Low-Voltage Differential Signalling. No external components needed. Common-mode voltage is programmable via VCM. Most FPGAs accept LVDS natively.
  • CML — Current Mode Logic. Requires 50 Ω pull-up resistors to VCC on the board. Higher bandwidth but more board complexity.
Input PWR: 5 (logic clock drive strength)
Sets the LOGICLKOUT0 driver output power (LOGICLK_PWR), range 0–7. Codes 0–7 map to output swings from ~0.1 V to ~0.5 V single-ended peak-to-peak in LVDS mode. PWR = 5 gives approximately 0.35 V single-ended swing — a typical standard LVDS level. Higher PWR increases swing but also current draw (~85 mA total for the LOGICLK section).
Input VCM: 2 (common-mode voltage)
Sets the LVDS common-mode voltage of LOGICLKOUT0 (LOGICLK_VCM). Code 2 corresponds to approximately 1.18 V common-mode. TICS Pro shows the valid VCM range for each PWR setting (some codes are invalid for certain PWR levels and are greyed out). Match this to your FPGA's LVDS input common-mode specification — usually 0.9–1.25 V.
Input FMT / PWR / VCM (LOGISYSREF)
A second, independent set of format, power, and VCM fields for the LOGISYSREFOUT output. When LOGICLK2_EN is off, this pin carries the SYSREF companion for LOGICLKOUT0. The format (LVDS shown), power (5), and VCM (2) are configured identically to the main LOGICLK here — but they can be set differently if your SYSREF receiver has different requirements than your clock receiver.
Input Edge / Min / DLY (logic clock delay)
Same controls as Block 5 but for the logic clock path:
  • Edge — selects which clock edge the output is aligned to.
  • Min — snaps to the minimum valid delay code.
  • DLY: 0 — the per-output delay code for LOGICLKOUT0. The propagation delay shown (2708.66 ps) is the same as the main channel — both are derived from the same CLKIN, so their absolute delays are comparable.
Calculated LOGICLK 16.0 MHz + LOGICLK_EN
The calculated LOGICLKOUT0 frequency (16 MHz) is displayed next to the output buffer enable checkbox (LOGICLK_EN). Check LOGICLK_EN to actually drive the pin — even if LOGIC_EN is on, the output buffer is separately gated so you can keep the divider chain running during SYNC without glitching the FPGA clock.
Calculated LOGISYSREF 16.0 MHz + LOGISYSREF_EN
LOGISYSREF frequency (same as SYSREF divider → 16 MHz) and its output buffer enable. In most JESD204C designs, LOGISYSREF should run at the same frequency as the main SYSREF outputs so the FPGA and the ADC/DAC all receive the same synchronisation heartbeat and can all participate in the same deterministic-latency alignment procedure.
Putting It All Together With CLKIN = 1280 MHz and the settings shown across all six blocks: CLKOUT0–3 each output 1280 MHz (buffer mode) to four ADC/DAC chips; LOGICLKOUT0 outputs 16 MHz to the FPGA; all five SYSREF outputs fire at 16 MHz, aligned to within ≈24 ps of each other via the phase interpolators. One SYNC pulse from the FPGA aligns all dividers simultaneously. One SYSREF pulse then latches every converter into Subclass 1 deterministic latency. The whole clock tree is now phase-coherent from 1280 MHz DAC samples down to the 16 MHz FPGA frame clock.
