Joint BLE & mioty Localisation Module

A coin-cell BLE + mioty localization tag combining precise short-range tracking with long-range fallback — one 2.4 GHz transceiver, motion-gated power, and a 9-axis IMU on a custom circular PCB.

● Ongoing · Started Jan 2025 · FAU Erlangen-Nürnberg · M.Sc. Research Project
Overview

What is it?

The Joint BLE & mioty Localization Module is a compact, low-power tag designed to track assets indoors and outdoors using two complementary wireless protocols — Bluetooth Low Energy (BLE) and mioty — sharing a single 2.4 GHz SX1280 transceiver.

BLE advertising packets enable precise short-range localization via RSSI or AoA/AoD when anchors are available. If no anchor is in range, the module seamlessly falls back to mioty's telegram splitting access scheme for long-range, low-power reporting. Both operate in the 2.4 GHz ISM band, eliminating the need for a second radio and keeping the coin-cell bill of materials tight.

Motivation

Why one transceiver for both?

01
Bill of materials. A single SX1280 handles both BLE and mioty, cutting component count and PCB area compared to two-radio designs.
02
Power budget. The nRF52840 MCU gates radio power via the IMU's motion interrupt — the radio is off when the asset is stationary, extending coin-cell life significantly.
03
Coverage continuity. BLE gives high accuracy indoors; mioty keeps reporting alive outdoors or in sparse deployments where BLE anchors don't reach.
04
Circular form factor. A round PCB matches common badge or label shapes, making the tag easy to embed in existing asset tracking housings.

Hardware

PCB design In progress

The board is a circular two-layer PCB designed in KiCad. The round form factor allows the tag to fit inside standard badge holders or circular asset labels. Key design constraints:

  • Diameter ≤ 40 mm to fit standard badge holders
  • Keep-out zone around the chip antenna to preserve radiation pattern
  • Crystal placed close to the MCU with dedicated ground pour
  • CR2032 holder on the back side, LDO and decoupling caps on front
  • SWD debug header on the board edge (4-pin, 1.27 mm pitch)

Firmware

Software architecture In progress

The firmware runs on Zephyr RTOS targeting the nRF52840. It is split into three cooperative threads: a sensor thread reading the IMU, a radio thread managing BLE advertising and mioty transmission, and a power management thread handling sleep transitions.

  • IMU driver — SPI driver for ICM-42688, configures motion-wakeup interrupt threshold
  • BLE stack — Zephyr's host stack, advertising with custom manufacturer data containing RSSI-based location hint
  • mioty layer — SX1280 in FLRC mode at 1.3 Mbit/s, applying telegram splitting for mioty compatibility
  • Power state machine — active on motion, deep sleep (<10 µA) when stationary for >30 s
  • Web interface — BLE scanner running in browser (Web Bluetooth API) visualizes real-time position on a floor plan
Localization

Algorithm Planned

Short-range localization uses RSSI trilateration from fixed BLE anchors. Each anchor reports RSSI to a gateway; the gateway runs a weighted least-squares solver with Kalman filtering to smooth position estimates. Anchor positions are stored in a JSON config file served alongside the web interface.

When the asset is out of BLE range, mioty telemetry packets carry the last known IMU orientation and a dead-reckoning delta, allowing the gateway to maintain an approximate position estimate until BLE coverage resumes.
